02

Painel de configuração do card

Botão “⚙ Card” na barra do Kanban abre este painel. Presets no topo (Compacto / Padrão / Completo) aplicam um conjunto de toggles de uma vez; abaixo, cada campo é um interruptor. Salvo por projeto.

Como funciona:

  • Presets = atalhos. “Compacto” desliga descrição/progresso/tags; “Completo” liga tudo. Mexer num toggle individual muda o preset para “Personalizado”.
  • Alguns campos são sempre visíveis e não entram como toggle: título, código EAP, ícone de tipo, bloqueio e alerta de subs impedidas (sinais críticos).
  • Vale por projeto — cada quadro lembra sua configuração (localStorage), igual densidade e filtros.
  • Substitui a limitação atual: hoje a densidade “S” apaga todos os badges de uma vez, sem escolha. Aqui você decide item a item.
Decisão em aberto: manter os 3 presets + toggles finos (recomendado), ou só os toggles finos sem preset? Ver seção 04.

O que preciso que você decida

  • Layout base: aprova o card “limpo” da seção 01 (uma linha de chips, código no título, avatar sem nome) como novo padrão — ou prefere manter o layout atual e só adicionar os toggles por cima?
  • Presets: manter os 3 presets (Compacto/Padrão/Completo) + toggles finos, ou só os toggles?
  • Defaults: concorda com a coluna “Default” da seção 03 (descrição, breadcrumb, resumo de subs, tags e dependências desligados por padrão)? Ajusto o que quiser antes de codar.
  • Onde fica o botão: um botão “⚙ Card” separado na barra, ou dentro do mesmo menu onde hoje está a densidade S/M/G?

Tudo isto é front-end puro (sem migration). Reaproveita a cadeia de props da densidade e uma nova chave kanban-card-fields:{projeto}. Depois que você decidir, implemento sem commit.
